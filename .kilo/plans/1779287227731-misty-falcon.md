# Fix: DropZone File Picker Click Not Opening Native File Explorer

## Root Cause

The DropZone component (`src/components/DropZone.tsx`) uses an invisible (`opacity-0`) absolutely-positioned file input overlay to intercept clicks. This implicit CSS-stacking-based approach has two critical issues:

1. **Click does not reliably open the file dialog** — The component has no explicit `onClick` to trigger the hidden input. It relies entirely on CSS paint order for the invisible input to intercept clicks. While CSS spec says positioned elements (layer 6) paint above in-flow elements (layer 3), this breaks in edge cases involving flex containers, browser rendering quirks, and `opacity:0` file input handling inconsistencies across Chrome/Edge/Firefox.

2. **No keyboard accessibility** — The invisible input has no visible focus indicator, no `tabIndex` management, and no keyboard handlers. Enter/Space on the drop zone do nothing.

## Fix Strategy

Replace the implicit stacking-based overlay approach with an explicit `inputRef.current?.click()` pattern triggered by `onClick` and `onKeyDown` on the container div. This is the battle-tested pattern used by production file upload components.

### Key changes in `src/components/DropZone.tsx`:

1. **Add `useRef` import and `inputRef`** — ref to programmatically click the hidden input
2. **Add `handleClick` callback** — calls `inputRef.current?.click()` when container is clicked
3. **Add `handleKeyDown` callback** — triggers file picker on Enter/Space for keyboard accessibility
4. **Wire up container div** — add `onClick`, `onKeyDown`, `tabIndex={0}`, `role="button"`, `aria-label`
5. **Change input from invisible overlay to sr-only** — input no longer needs to cover the area; `sr-only` keeps it accessible while hiding it visually, and `.click()` works reliably on it
6. **Add `ref={inputRef}` and `tabIndex={-1}` to the input** — prevents double-tabbing (the container is the focus target)
7. **Remove `e.preventDefault()` from `handleChange`** — change event is not cancelable per spec; cleanup only

### Why this is robust:

- **Click reliability**: Explicit `onClick → inputRef.click()` bypasses all CSS stacking concerns
- **Keyboard accessibility**: `tabIndex={0}` + `role="button"` + `onKeyDown` enables Enter/Space
- **Cross-browser**: `.click()` on hidden file inputs works in Chrome, Edge (Chromium), Firefox, and Safari
- **Drag & drop preserved**: No changes to drag handlers or `handleDrop`; drag and click are independent event flows
- **UI unchanged**: Container CSS remains identical; only the input is moved from overlay to sr-only
- **No hacks**: Standard React pattern using refs and event handlers

## Verification

- Click anywhere on "Select or drop file" area → native file explorer opens
- Drag and drop a file → file is selected (existing behavior preserved)
- Tab to the drop zone area → focus ring visible; Enter/Space opens file explorer
- Works on Windows Chrome, Edge, Firefox
